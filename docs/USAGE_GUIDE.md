## 1. インストール

### 1-1. ソースコードを入手

```bash
git clone https://github.com/katu09161004/youtube-article-extension.git
```

または GitHub からZIPダウンロード → 展開

### 1-2. Chromeに読み込み

1. Chrome のアドレスバーに `chrome://extensions` と入力
2. 右上の「**デベロッパーモード**」トグルを **ON** にする
3. 「**パッケージ化されていない拡張機能を読み込む**」ボタンをクリック
4. ダウンロードした `src/` フォルダを選択
5. 拡張機能一覧に「YouTube Article Generator」が表示されればOK

> **ヒント**: 拡張機能アイコンをツールバーにピン留めしておくと便利です。
> パズルマーク → YouTube Article Generator の横のピンアイコンをクリック

## 2. 初期設定（APIキー）

### 2-1. 設定画面を開く

以下のどちらかで開きます:

- **方法A**: ツールバーの拡張アイコンをクリック → ポップアップの「**設定**」リンク
- **方法B**: `chrome://extensions` → YouTube Article Generator の「**詳細**」→「**拡張機能のオプション**」

### 2-2. GitHub設定

| 項目 | 入力内容 | 例 |
|:--|:--|:--|
| GitHub Personal Access Token | GitHubの個人アクセストークン | `ghp_xxxxxxxxxxxx` |
| GitHub Repository | 記事を保存するリポジトリ | `katu09161004/LIFE` |

> **トークンの取得方法**: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → `repo` 権限にチェック

### 2-3. AI APIキー設定

最低1つのAI APIキーが必要です。上から順に試行され、最初に成功したものが使われます（フォールバック）。

| 優先順 | プロバイダ | モデル | 入力欄 |
|:--:|:--|:--|:--|
| 1 | さくらのAI Engine | gpt-oss-120b | さくらAI Token |
| 2 | Google Gemini | Gemini 1.5 Pro | Gemini API Key |
| 3 | OpenAI | GPT-4o | OpenAI API Key |
| 4 | Anthropic | Claude | Anthropic API Key |

### 2-4. デフォルト記事設定（任意）

記事生成モーダルの初期値を設定できます:

| 項目 | 説明 | デフォルト |
|:--|:--|:--|
| プラットフォーム | 投稿先 | LIFE |
| 難易度 | 記事の難易度 | 中級者向け |
| 対象読者 | 想定読者 | エンジニア全般 |
| スタイル | 記事のスタイル | 自動判定 |

### 2-5. 保存

「**保存**」ボタンをクリック → 「設定を保存しました」と表示されタブが自動で閉じます。

## 3. 記事を生成する

### 3-1. YouTube動画を開く

Chromeで YouTube の動画ページ（`youtube.com/watch?v=...`）を開きます。

### 3-2. 「記事化」ボタンをクリック

動画プレイヤーの下、いいね・共有ボタンの横に青い「**記事化**」ボタンが表示されます。

> **ボタンが表示されない場合**: ページを再読み込み（Ctrl+R / Cmd+R）してください。
> YouTube はSPA（シングルページアプリ）のため、別の動画からの遷移時は数秒かかることがあります。

### 3-3. モーダルで設定を選択

ボタンをクリックすると設定モーダルが表示されます:

| 設定項目 | 説明 |
|:--|:--|
| **投稿先プラットフォーム** | 記事のフォーマットを決定（下記参照） |
| **スタイル** | 「自動判定」推奨。料理動画は自動でレシピモードに切り替わります |
| **難易度** | 初心者向け〜専門家向け |
| **対象読者** | 自由入力（例: Pythonエンジニア、料理初心者） |
| **記事タイトル** | 空欄ならAIが自動生成。手動指定も可 |

#### プラットフォーム別の違い

| プラットフォーム | 用途 | 特徴 |
|:--|:--|:--|
| **LIFE** | 個人ナレッジ保存 | GitHubに蓄積。後で別形式に再利用可 |
| **Zenn** | 技術記事投稿 | YAML frontmatter付き。push連携で自動公開可 |
| **Qiita** | 技術記事投稿 | Qiita用frontmatter付き |
| **note** | エッセイ・料理・教養 | noteメタ情報コメント付き |
| **Cookpad** | 料理レシピ専用 | 材料/作り方/コツを分割出力。スクショガイド付き |

### 3-4. 「記事を生成」をクリック

処理の流れ:

```
字幕を取得中...        → 動画の字幕データを取得
  ↓
AI記事を生成中...      → AIが記事を執筆（30秒〜2分）
  ↓
GitHubに保存中...      → LIFEリポジトリに自動保存
  ↓
記事化完了!
```

## 4. 生成した記事を使う

### 完了画面でできること

| ボタン | 動作 |
|:--|:--|
| **GitHubで記事を見る** | GitHubの記事ページが新しいタブで開く |
| **記事をコピー** | 記事全文がクリップボードにコピーされる |
| **閉じる** | モーダルを閉じる |

### GitHubでの保存先

記事は以下のパスに自動保存されます:

```
LIFE/{ジャンル}/{プラットフォーム}/YYYYMMDD_HHMM_{VideoID}_{タイトル}.md
```

例:
```
LIFE/Tech/Zenn/20260305_1430_dQw4w9WgXcQ_PythonでMCPサーバーを作る.md
LIFE/Cooking/note/20260305_1500_abc123xyz_本格カルボナーラの作り方.md
```

## 5. 各プラットフォームへの投稿方法

### Zenn

1. GitHubリポジトリをZennと連携（zenn.dev → 設定）
2. 記事のfrontmatterで `published: true` に変更してpush → 自動公開
3. または zenn.dev エディタに記事を直接貼り付け

### Qiita

1. qiita.com → 「記事を書く」 → Markdownモード
2. frontmatterのYAML部分は削除してから貼り付け
3. タグ・タイトルはQiitaのUI上で設定

### note

1. note.com → 投稿 → テキスト
2. 記事冒頭の `<!-- note... -->` コメントを参考にタイトル・ハッシュタグを設定
3. 画像はサムネイルURLを貼り付け or スクリーンショットをアップロード

### Cookpad

1. cookpad.com/recipe/new を開く
2. 「`=== COOKPAD:料理名 ===`」の内容を「料理の名前」欄に貼る
3. 「`=== COOKPAD:材料 ===`」の内容を「材料」欄に貼る（タブ区切りで自動2列化）
4. 「`=== COOKPAD:作り方 ===`」の各ステップを工程欄に1つずつ貼る
5. 「`=== COOKPAD:コツ・ポイント ===`」を対応欄に貼る
6. スクショガイドのタイミングで動画を一時停止 → スクショ撮影 → 工程写真としてアップロード

## 6. ポップアップの使い方

ツールバーの拡張アイコンをクリックすると表示されます:

| 表示項目 | 内容 |
|:--|:--|
| **現在の動画** | YouTubeの動画ページにいる場合、動画タイトルとIDを表示 |
| **最近の記事** | 直近5件の生成記事リスト（クリックでGitHubを開く） |
| **設定** | 設定画面を開く |
| **GitHub LIFE** | LIFEリポジトリをブラウザで開く |
| **API状態** | GitHub・AI APIキーの設定状況（●=設定済み、○=未設定） |

## 7. 設定項目の詳細

### ジャンル自動判定

字幕テキストからジャンルを自動判定します:

| ジャンル | 判定キーワード例 |
|:--|:--|
| **Tech** | python, ai, llm, docker, aws, プログラム, 開発 |
| **Cooking** | recipe, 料理, レシピ, 食材, 作り方, パスタ |
| **Medical** | medical, 医療, 看護, 診断, 治療 |
| **Science** | science, 科学, 物理, 宇宙, 研究 |
| **Business** | business, 投資, 経営, マーケティング |
| **Other** | 上記に該当しない場合 |

### 料理モード自動切替

字幕に料理キーワードが2つ以上含まれると自動的に**料理レシピモード**に切り替わります。
手動で切り替えたい場合は、スタイルを「レシピ記事」に変更してください。

## 8. トラブルシューティング

### 「記事化」ボタンが表示されない

- ページを再読み込みしてください（Ctrl+R / Cmd+R）
- `chrome://extensions` で拡張が有効になっているか確認
- YouTubeの動画ページ（`/watch?v=...`）でのみ表示されます

### 字幕取得に失敗する

- 字幕のない動画では使用できません
- ライブ配信やプレミア公開中の動画は非対応です
- 自動生成字幕が無効になっている動画もあります

### AI記事生成に失敗する

- 設定画面でAI APIキーが正しく入力されているか確認
- 複数のAPIキーを設定しておくとフォールバックが効きます
- APIの利用制限（レート制限）に達している可能性があります

### GitHub保存に失敗する

- GitHub Token に `repo` 権限があるか確認
- リポジトリ名（`username/LIFE`）が正しいか確認
- LIFEリポジトリ内に対象フォルダ（`LIFE/Tech/Zenn/` 等）が存在するか確認

### 拡張機能の更新方法

コードを更新した後:
1. `chrome://extensions` を開く
2. YouTube Article Generator の更新ボタン（丸い矢印）をクリック
3. YouTubeのタブを再読み込み
